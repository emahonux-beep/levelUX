Working Efficiently in a Page

schedule 6 min read
🎬 On-screen: title card, "Working Efficiently in a Page"

You know the toolbar now. This lesson is about the habits and shortcuts that separate a fast, findable file from one where every task takes twice as long as it should.

Sectioning and naming

We touched on this in Lesson 3, but it's worth repeating here because it's the foundation everything else builds on:

  • Section your pages. Group related frames so someone can scan a page and understand its shape in seconds.
  • Name your frames and layers deliberately. A layer called "Group 14" is a dead end. A layer called "Nav — Mobile — Collapsed" tells the next person exactly what they're looking at.
  • One frame per full page, with sections beneath it for individual component variants — instead of duplicating the whole page for every small state change.

If you have to ask "what is this layer" more than once, the name failed. Fix it the moment you notice, not later.

Paste, replace, and find

🎬 On-screen: side-by-side demo — Paste vs. Paste to Replace on the same component
  • Paste to replace swaps the content of a selected layer with what's on your clipboard, keeping the original layer's position and properties. Use this when you want to drop new content into an existing structure.
  • Paste over a section is closer to a standard paste, layering new content into a section rather than replacing something specific inside it.
  • Find and replace works the way you'd expect from any text tool, but on a Figma file it's a fast way to update repeated copy — like a product name — across dozens of frames at once.
  • Select all or select matching layers lets you grab every instance of a given layer type or name across a frame or page, instead of clicking each one individually. Genuinely useful when you need to make the same small change in twenty places.

Views and layout aids

A few tools that make dense pages easier to work in:

  • Layout guides and grids — keep spacing and alignment consistent as you build.
  • Slots — placeholder areas that make swapping content in and out of a structure faster.
  • Masks — clip one shape or image to the bounds of another, useful for anything from rounded thumbnails to more complex crops.
  • Vectors — for custom shapes and icon work beyond the basic shape tool.
  • Design panel vs. Prototype panel — the design panel is where you set properties (size, color, spacing); the prototype panel is where you wire up interactions and flow between frames. Knowing which one you're in saves you from hunting for a setting that lives in the other.

Show and hide

Small but genuinely useful toggles: you can show or hide the UI chrome to get a clean, distraction-free view of the canvas, and you can show or hide comments when you want to review a design without pin clutter in the way.

Exporting

You can export frames directly to PDF when you need something outside the Figma ecosystem — handy for stakeholder reviews or printed handouts.
