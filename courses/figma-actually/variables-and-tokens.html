Module 3 · Systems Thinking · Lesson 6 of 10

Variables & Tokens

schedule 6 min read
🎬 On-screen: title card, "Variables & Tokens"

This is the lesson where things start to feel less like decorating a canvas and more like building a system. If you only take one thing from this course, make it this one — variables are the difference between a file that scales and a file you're manually patching forever.

What variables actually do for you

Variables let you define a value once — a color, a piece of text, a spacing amount — and reuse it everywhere, so a single update propagates across the whole file instead of requiring you to hunt down every instance by hand.

🎬 On-screen: change one color variable, watch it update across a dozen frames at once

The main types you'll work with:

  • Color tokens — your palette, defined once and referenced everywhere.
  • Text tokens — headings, body copy, labels — consistent type across every screen.
  • Spacing tokens — consistent padding and margins instead of eyeballing "close enough."
  • Radius tokens — consistent corner rounding across every card, button, and input.
  • Content variables — swappable text content, which is where things get genuinely powerful.

The grocery store example

🎬 On-screen: build this live — one component, then swap the variable set

Say you're building a location card for a grocery store. It needs:

  • A heading — the store name
  • A body — the address
  • Three calls to action — call, website, get directions

Instead of duplicating that card for every store and manually retyping each field, you build it once and create variable sets for each store — ShopRite, Walmart, Publix. Flip between them with a click, and the whole card updates. No retyping the same content structure over and over.

Variables aren't just a time-saver. They're what makes consistency possible at scale — the moment you're maintaining more than a handful of screens by hand, something is going to drift.

Setting variables up in a file

You don't need an enterprise design system to benefit from this. Even in a single project file, it's worth setting up a basic set of color, text, spacing, and radius variables early — before you've built forty frames you'd otherwise have to go back and reconcile.

A plugin worth knowing

If you're working with a lot of variable data — think a big list of content variants, like our grocery store example scaled up to fifty locations — a CSV-to-variables plugin lets you import that data directly instead of entering it one field at a time. If you're the type who nerds out on variables, this is where it gets genuinely fun.
