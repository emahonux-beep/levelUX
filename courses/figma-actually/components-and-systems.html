Module 3 · Systems Thinking · Lesson 7 of 10

Components & Systems

schedule 5 min read
🎬 On-screen: title card, "Components & Systems"

Variables give you consistent values. Components are where those values actually get assembled into something reusable. This lesson covers building them, nesting them, and keeping them maintained once other people are relying on them too.

Auto layout first

🎬 On-screen: build a simple button with auto layout, then resize its text to show it adapt

Auto layout is what makes a component behave like real UI instead of a static picture of UI — padding, spacing, and alignment that respond automatically when content changes. If you're building anything you intend to reuse, start with auto layout before you start with a component. It's much harder to retrofit later than to build in from the start.

Local components

A local component lives inside a single file — not yet part of a shared library, just reusable within the project you're working on. This is usually where you start: build the thing, get it right, prove it out in context.

Components inside components

Once you're comfortable with basic components, the next step is nesting — a component that contains other components. Think of a card component that's built from a button component, an image component, and a text component. Each piece stays independently editable, but they assemble into something bigger and stay in sync when any one piece updates.

A well-nested component system means fixing something once, in one place, instead of hunting it down across every screen it appears on.

Publishing a component

When a local component is ready to be shared beyond a single file, you publish it to a library. That makes it available to anyone with access to that library, referenced through the Assets panel we covered in Lesson 4.

Updating a library

🎬 On-screen: make a change to a published component, show the update notification appear in a different file

Publishing isn't a one-time event. When you update a published component, anyone using it gets notified and can accept the update — which is exactly the mechanism that makes design systems worth maintaining in the first place. This is also where team discipline matters: publish updates deliberately, not accidentally, since a careless change can ripple into files you don't have visibility into.

What's next

Lesson 8 covers some of the more advanced tools available on the canvas — masks, vectors, layout guides, and the design and prototype panels in more depth.
