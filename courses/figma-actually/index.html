Module 1 · Foundations · Lesson 1 of 10

The Figma Ecosystem

schedule 4 min read

You've probably used more than one design tool to get here. Sketch, XD, maybe Axure back when prototyping tools were still figuring themselves out. Over the last few years, one platform pulled ahead and stayed there. That's what this lesson — and honestly this whole course — is about.

🎬 On-screen: title card, "The Figma Ecosystem"

Figma isn't one tool anymore. It's a suite, and knowing which piece to reach for is the first skill worth building. Here's the map.

What's actually in Figma today

🎬 On-screen: show the file picker with all the product tiles
ToolWhat it's for
editDesign
The canvas. Where UI actually gets built.
sticky_note_2FigJam
Whiteboarding, workshops, low-fidelity thinking out loud.
slideshowSlides
Presentations, built the same way you'd build anything else in Figma.
auto_awesomeMake
AI-assisted prototyping — describe something, get a working starting point.
campaignBuzz
Templated content production, mostly for marketing teams.
languageSites
Publish a working website straight from Figma.

That's six products. You will not live in all six. Most of your working week is going to happen in four places: Design, FigJam, Slides, and Make. The rest are good to know exist — you'll reach for them occasionally — but they're not where the craft happens.

Knowing the whole map matters less than knowing where you actually live. Spend your energy there first.

Where you'll actually spend your time

Design is the canvas — the file type most people mean when they say "Figma." This is where components, variables, auto layout, and dev mode all live. It's the one this course spends the most time on.

FigJam is the whiteboard. Lower stakes, higher speed. Sticky notes, user flows, research synthesis, workshop facilitation. If Design is where you build the thing, FigJam is where you figure out what the thing should be.

Slides behaves the way you'd expect if you've ever built a frame in Figma — same logic, same shortcuts, just packaged for presenting instead of prototyping.

Make is the newest of the four, and it changes the workflow that used to require jumping out to a separate AI tool. You can prompt for a starting point right where you're already working, then bring it back to the canvas to refine by hand.

Why this matters before you touch a single frame

New designers — and honestly, some experienced ones — waste real time defaulting to the wrong tool. Mapping out a rough user flow inside a high-fidelity Design file. Running a workshop with no sticky notes because nobody thought to open FigJam. Building a five-slide deck frame by frame in Design when Slides already does that job.

🎬 On-screen: quick split-screen — same task done in the "wrong" tool vs. the "right" tool

Picking the right tool for the stage of work you're in isn't a small thing. It's the difference between moving fast and fighting your tool the whole way.
